    FAQ — Питання та відповіді

    Навіщо так багато тегів?

    Бо жанр не описує книгу повністю. Теги дозволяють передати: - теми - мотиви - настрій - тип сюжету

    Чи можна додавати нові теги в майбутньому?

    Так. Система спроєктована так, що: - нові теги не потребують змін у коді - вони автоматично зʼявляються у фільтрах

    Чим slug відрізняється від назви тегу?

    • Name — для людини
    • Slug — для URL

    Slug стабільний і не змінюється при перейменуванні тегу.

    Чому немає ієрархії тегів?

    Ієрархія: - ускладнює модель - погано працює для літератури

    Комбінація тегів дає більшу гнучкість.

    Як працює фільтрація книг?

    • OR — книга має хоча б один обраний тег
    • AND — книга має всі обрані теги

    Логіка реалізована на рівні БД через ORM.

    Як працюють “схожі книги”?

    Схожість визначається кількістю спільних тегів. Чим більше перетин — тим вище книга в списку.

    Чи потрібен machine learning?

    Ні. На цьому етапі прості й пояснювані правила: - працюють швидко - легко підтримуються - зрозумілі людині

    Чи можна зробити API або Telegram-бота?

    Так. Вся бізнес-логіка вже відокремлена й готова до використання через API.

    Чи можна масштабувати систему?

    Так. - PostgreSQL - нормалізована модель - відсутність жорстких залежностей

    Система готова до росту каталогу.
