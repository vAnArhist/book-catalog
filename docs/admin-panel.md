# Адмін-панель

Адмін-панель використовується для повного управління каталогом книг
і є основним інтерфейсом наповнення бази даних.

Побудована на стандартній Django Admin.

---

## Доступ

Адмінка доступна за адресою:

/admin/


Вхід дозволений тільки користувачам зі статусом `staff`.

---

## Управління книгами

### Створення книги
У формі створення книги можна:
- вказати назву, опис, рік, ISBN
- вибрати одного або кількох авторів
- додати теги (з автодоповненням)
- завантажити обкладинку та скани

---

### Теги в адмінці

Поле `Tags` має **автодоповнення**:
- при наборі тексту показуються існуючі теги
- це запобігає дублюванню

Рекомендовано:
- використовувати CAPSLOCK для “великих” категорій (ФАНТАСТИКА)
- звичайний регістр для сюжетних або тематичних тегів

---

### Зображення книги

Зображення додаються через inline-форму:
- `cover` — обкладинка
- `scan` — сторінки / ілюстрації

Сканів може бути кілька.

---

## Управління тегами

Теги можна:
- додавати
- перейменовувати
- переглядати slug

Slug генерується автоматично і використовується для URL.

---

## Кращі практики

- ❗ не створювати дублікати тегів
- ❗ не змінювати slug без потреби
- ✔ використовувати теги для опису **сенсу**, а не формату

---

## Висновок

Адмін-панель — це:
- джерело правди для каталогу
- інструмент для системного наповнення
- основа для якісного підбору книг
